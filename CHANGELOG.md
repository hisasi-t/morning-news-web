## 2026-05-16 cronを混雑回避時刻に変更・予備実行を追加

**背景:**
- 毎朝6:30 JSTに設定していた自動実行が、毎日10時間以上遅延していた
- 実際の実行時刻が5/12〜5/15すべて夕方17時前後で、もはや「朝刊」ではない状態
- GitHub Actions無料枠は0/15/30/45分のキューが混雑する仕様

**変更:**
- `cron: "30 21 * * *"` → `cron: "37 21 * * *"` に変更（21:37 UTC = 06:37 JST）
- 予備として `cron: "17 22 * * *"`（07:17 JST）を追加
- 2回目が走っても、内容に差分がなければ何もせず終わる設計

**今後の観察:**
- 1週間ほど実行時刻を見て、改善しなければC案（夕刊として割り切る）か他サービスへの移行を検討

## 2026-05-08 日付をJSTで生成するよう修正（根本バグ）

**背景:**
- 「更新ボタンを押しても日付が昨日のままになる」とひさしさんから報告
- 当初はiOSのキャッシュ問題と判断 → だが本当の原因は別だった
- GitHub ActionsがUTCで動くため、`datetime.datetime.now()`がUTC時刻を返していた
- 結果、日本では朝なのにヘッダーに「昨日の日付」が表示される状態だった

**変更:**
```python
JST = datetime.timezone(datetime.timedelta(hours=9))
now = datetime.datetime.now(JST)
```

**学び:**
- 表面的な症状（キャッシュ）だけでなく、根本原因を疑うべきだった
- ひさしさんの「更新ボタン押しても直らない」という違和感が根本原因にたどり着く手がかりになった

## 2026-04-29 ホーム画面アプリのキャッシュ問題対策

**背景:**
- iPhoneのホーム画面に追加したWebアプリで、古いHTMLがキャッシュされて表示される
- iOSはホーム画面アプリのHTMLを強くキャッシュする仕様

**変更:**
- `Cache-Control: no-cache` / `Pragma: no-cache` / `Expires: 0` メタタグ追加
- `apple-mobile-web-app-capable` 等のメタタグでホーム画面アプリ最適化
- ヘッダーに**手動「更新」ボタン**を追加
- `visibilitychange` / `pageshow` イベントで30分以上古ければ自動再読み込み
- ビルド時刻をJSのBUILD_TS定数として埋め込み、経過時間判定に使用

## 2026-04-27 初回リリース：Web版朝刊エージェント

**背景:**
- ローカル版（`~/Desktop/無料morning_news/`）はMacが起動してないと見れない

- 公開URL: https://hisasi-t.github.io/morning-news-web/

**設計方針:**
- モノトーン・絵文字なし・要約付き

- フィードはRSSのみ（APIキー不要、完全無料）
- カテゴリ: A=大工・建築 / B=AI・Claude / C=個人開発 / D=知的生産・Obsidian / E=相棒AIカルチャー
